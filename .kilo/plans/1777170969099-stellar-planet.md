## Emotion Detection (client-side, pre-API)

| Keyword group | Keywords | emotion value |
|---|---|---|
| Anxious | doob jayega, kho jayega, dar, safe hai kya, khatra | `anxious` |
| Confused | kya matlab, samajh nahi, kaisa, explain, batao | `confused` |
| Excited | badhiya, sahi, achha, mast, zordaar | `excited` |
| Default | — | `neutral` |

Pass `emotionState` in POST body.

## API Design

### `POST /api/chat`
**Request body:**
```json
{
  "message": "string",
  "language": "hi|bn|ta|mr|as|bho|raj|mai|kn|te|ml|en",
  "emotionState": "anxious|confused|excited|neutral",
  "chatHistory": [{"role": "user"|"assistant", "content": "string"}]
}
```

**Response:** `ReadableStream` — stream tokens as they arrive (Content-Type: text/event-stream or application/x-ndjson).

**System prompt template:**
```
You are VAANI, a warm rural Indian finance advisor. Language: {language}. Emotion detected: {emotion}.
Rules:
- Respond in {language} dialect (use Devanagari script for Hindi/Bhojpuri/Rajasthani/Maithili, Bengali script for Bengali, Tamil script for Tamil, etc.)
- Use village analogies when emotion=anxious (e.g. "paisa galla mein band hai jaise darwaza")
- Suggest specific FD products from Suryoday 9.1%, Utkarsh 8.5%, Jana 8.25% with real rates
- Always end with a reassuring phrase (e.g. "Chinta mat kar, hum hain na")
- Never use English banking jargon
- Give rupee examples specific to rural India (e.g. "₹5000 se shuru karein")
- When emotion=excited, give precise numbers and bold suggestions
- When emotion=confused, explain slowly with examples
```
