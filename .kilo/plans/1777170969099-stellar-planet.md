# Accessibility Cleanup Plan

## Files to Inspect
All React components: `src/**/*.jsx`

## Findings

### 1. Skip-to-content link — REMOVE
- **File:** `src/components/ChatWindow.jsx:197-208`
- **Issue:** `<a href="#chat-messages">Skip to messages</a>` — visible on focus, creates clutter
- **Action:** Delete entirely

### 2. Redundant `title` attribute — REMOVE
- **File:** `src/components/MessageBubble.jsx:209`
- **Issue:** `title="सुनें"` on speak button — already has `aria-label="यह संदेश सुनें"` with visible icon + tooltip
- **Action:** Remove `title="सुनें"` attribute

### 3. Empty aria-live region — FIX
- **File:** `src/context/AccessibilityContext.jsx:75-78`
- **Issue:** `<div role="status" aria-live="assertive" aria-atomic="true" style={{ position: 'absolute', width: '1px', height: '1px', overflow: 'hidden', clip: 'rect(0,0,0,0)', whiteSpace: 'nowrap', border: 0 }}>` — always in DOM, creates 1x1px whitespace region even when empty (announcement is empty string by default)
- **Action:** Change to conditionally render only when `announcement` has content

### 4. aria-live regions to KEEP (semantic, have content)
- `ChatWindow.jsx:214` — `role="alert" aria-live="assertive"` on offline banner (has content when visible)
- `ChatWindow.jsx:226` — `role="alert" aria-live="assertive"` on error banner (has content when visible)
- `ChatWindow.jsx:313` — `role="log" aria-live="polite"` on messages container (has messages)
- `TypingIndicator.jsx:19` — `aria-live="polite" aria-label="Vaani सोच रहा है"` (has content)

### 5. Other patterns checked — NOT FOUND
- No `sr-only` or screen-reader-only text
- No skip links (except the one above)
- No `role="presentation"` wrapper divs
- No focus-ring outlines on non-keyboard elements
- No hidden tooltip divs
